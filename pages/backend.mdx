# Backend API

GoToKart backend is a **Spring Boot** application exposing REST APIs for authentication, users, products, shopping cart, and orders, backed by **MySQL** and secured with **JWT**.

## Stack

- Java 21 · Spring Boot 3.x · Spring Security · Spring Data JPA · JJWT 0.11.5 · BCrypt · Maven

## Base URL

`http://localhost:8080/api`

For deployed environments, replace the host with your production API domain.

## Authentication

GoToKart uses **JWT** tokens. Passwords are hashed with BCrypt.

### Default admin credentials

| Field | Value |
|-------|-------|
| Email | `admin@gotokart.com` |
| Password | `admin123` |
| Role | `ADMIN` |

The admin account is created automatically on backend startup if it does not exist.

### Using JWT in requests

After login, include the token in every admin request:

```
Authorization: Bearer <your-jwt-token>
```

### Auth endpoints

| Method | Path | Auth | Description |
|--------|------|:---:|-------------|
| `POST` | `/api/auth/login` | No | Login, returns JWT |
| `POST` | `/api/auth/register` | No | Register, returns JWT |
| `GET` | `/api/users/me` | JWT | Get current user profile |

<details>
<summary><strong>POST</strong> <code>/api/auth/login</code> — login</summary>

**Request body**

```json
{
  "email": "admin@gotokart.com",
  "password": "admin123"
}
```

**Response**

```json
{
  "token": "eyJhbGciOiJIUzI1NiJ9...",
  "id": 1,
  "name": "Admin",
  "email": "admin@gotokart.com",
  "role": "ADMIN"
}
```
</details>

<details>
<summary><strong>POST</strong> <code>/api/auth/register</code> — register new user</summary>

**Request body**

```json
{
  "name": "Ankit Pandey",
  "email": "ankit@gotokart.com",
  "password": "secret123"
}
```

**Response** — same shape as login response, with `role: "USER"`.
</details>

## Products

| Method | Path | Auth | Description |
|--------|------|:---:|-------------|
| `GET` | `/api/products` | No | List all products |
| `GET` | `/api/products/{id}` | No | Get product by ID |
| `POST` | `/api/products` | ADMIN JWT | Create product (merges stock if name exists) |
| `PUT` | `/api/products/{id}` | ADMIN JWT | Update product |
| `DELETE` | `/api/products/{id}` | ADMIN JWT | Delete product |

<details>
<summary><strong>POST</strong> <code>/api/products</code> — create product</summary>

**Request headers**

```
Authorization: Bearer <admin-jwt-token>
Content-Type: application/json
```

**Request body**

```json
{
  "name": "iPhone 15",
  "description": "Apple iPhone 15 128GB",
  "price": 79999.0,
  "stock": 50,
  "category": { "id": 1 }
}
```

**Helpful note**

- If a product with the same name already exists, backend merges/increases stock instead of creating a duplicate row.
- Returns `403 Forbidden` if the `Authorization` header is missing or the user is not `ADMIN`.
</details>

<details>
<summary><strong>DELETE</strong> <code>/api/products/&#123;id&#125;</code> — remove product</summary>

**Request headers**

```
Authorization: Bearer <admin-jwt-token>
```

**Helpful note**

- Ask for explicit confirmation in frontend before calling delete.
</details>

## Cart

| Method | Path | Auth | Description |
|--------|------|:---:|-------------|
| `GET` | `/api/cart/{userId}` | No | Cart for user |
| `POST` | `/api/cart/{userId}/add?productId=&quantity=` | No | Add item |
| `DELETE` | `/api/cart/{userId}/remove?productId=` | No | Remove item |

<details>
<summary><strong>POST</strong> <code>/api/cart/&#123;userId&#125;/add?productId=&amp;quantity=</code> — add to cart</summary>

**Example**

```text
/api/cart/1/add?productId=2&quantity=3
```

**Helpful note**

- Clamp quantity on the UI side so users cannot submit 0 or negative values.
</details>

## Database

Create a database named `gotokart` and configure JDBC URL, user, and password in `application.properties`. See the [Getting started](getting-started) guide for full examples.

On first startup, the backend automatically seeds **102 products** across 11 categories and creates the admin user.
