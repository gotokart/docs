# Getting started

## Clone the repositories

```bash
git clone https://github.com/gotokart/backend.git
git clone https://github.com/gotokart/frontend.git
git clone https://github.com/gotokart/infra.git
git clone https://github.com/gotokart/docs.git
```

## Prerequisites

- **Backend:** Java 21, Maven 3.9+, MySQL 8+
- **Frontend:** modern browser; backend running locally
- **Docs (this site):** Node.js 18+, npm (or pnpm/yarn)

---

## Option A — Run with Docker (recommended)

The simplest way to run everything locally is via Docker Compose from the `infra` repo:

```bash
cd infra
docker compose -f docker-compose.local.yml up --build
```

This starts **MySQL**, the **Spring Boot backend**, and **Nginx** (which serves the frontend at `http://localhost`).

---

## Option B — Run backend directly

### 1. Set up MySQL database

```sql
CREATE DATABASE gotokart;
```

### 2. Configure `application.properties`

Edit `backend/src/main/resources/application.properties`:

```properties
spring.datasource.url=jdbc:mysql://localhost:3306/gotokart?useSSL=false&allowPublicKeyRetrieval=true&serverTimezone=UTC
spring.datasource.username=root
spring.datasource.password=Root@1234

jwt.secret=GoToKartSuperSecretJwtKey2026!!VeryLongAndSecure@#$%
jwt.expiry-ms=86400000
```

### 3. Start the backend

```bash
cd backend
mvn clean spring-boot:run
```

API base URL: `http://localhost:8080`

### 4. Open the frontend

Open `frontend/index.html` directly in your browser. The UI calls `http://localhost:8080/api` by default (configured in `script.js`).

---

## Default admin account

On first startup, the backend automatically seeds **102 products** and creates an admin user:

| Field | Value |
|-------|-------|
| Email | `admin@gotokart.com` |
| Password | `admin123` |
| Role | `ADMIN` |

Log in with these credentials to access the admin panel (add/delete products).

---

## How authentication works

1. Click **Login** → enter email and password
2. The frontend calls `POST /api/auth/login` and receives a JWT token
3. The token is saved to `localStorage` and sent as `Authorization: Bearer <token>` on admin requests
4. Your session is automatically restored on page reload

## Run the documentation site locally

```bash
cd docs
npm install
npm run dev
```

Then open `http://localhost:3000` (or the URL printed in the terminal).
