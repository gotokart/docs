## High-level architecture

<div className="gk-arch-note">
  Request flow: Browser loads storefront, storefront calls backend APIs, backend persists data in MySQL.
</div>

<div className="gk-arch-grid">
  <div className="gk-arch-card">
    <strong>1) Presentation layer</strong>
    <span>Static storefront hosted on GitHub Pages serves UI, filters, cart, and order flow.</span>
  </div>
  <div className="gk-arch-card">
    <strong>2) API layer</strong>
    <span>Spring Boot backend exposes `/api/users`, `/api/products`, `/api/cart`, and `/api/orders`.</span>
  </div>
  <div className="gk-arch-card">
    <strong>3) Data layer</strong>
    <span>MySQL stores users, products, cart items, orders, and order item snapshots.</span>
  </div>
  <div className="gk-arch-card">
    <strong>4) Delivery layer</strong>
    <span>GitHub Actions builds docs and deploys updates to GitHub Pages automatically.</span>
  </div>
</div>
